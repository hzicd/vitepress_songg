# bug起源
今天打算是写一个日记，在浏览About me的时候，发现了元素堆叠顺序的bug，也就是说，我的图片上划到导航栏时候，导航栏应该是在图片上面，覆盖图片。但是，我发现是图片在导航栏的上面就显得很突兀
# 解决
我先问了ai，我觉得是元素z-index问题，但是我想印象我的导航栏的z-index属性给的很高，是1000还是9999。我觉得不应该是，因为我用的html语法插入的图片，用md语法去插入图片，反而就正常了。这时候我意识到可能是html插入图片的z-index属性，我给了负值。结果还是没有生效，我继续问了ai，ai说这与position属性有关，于是又去手机问了豆包。果然没错，就是z-index没生效，需要设置一个position为relative就行。这些是前端的知识了。
# 感悟和总结  

html中position和z-index有着密切联系。其实，很多东西就是先学一个小小的基础，后面的进阶，则是在做中学，而不是一味的死读书，这是有方法的。纸上谈兵，一直学习理论，缺乏实践是不可取的。但是，在前期学习，是可以的。后期的学习，再这样就很难进步，很鸡肋。东西是学不完的，后期，只有是在干中学才能有所成就。
