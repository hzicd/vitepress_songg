# 吐槽ai
起因是每次build打包vitepress项目的时候，我会发现有很多图片直接就是放在根目录下面，就导致显得非常冗余。  
![](/img/debug2.png)
这是25年的11月，我是想着用ai帮我解决这个重复性问题，因为我需要在很多文档文件里面，只要涉及插入图片的md语法，都要改路径前面加一个/img/即可。我把这个活交给ai了。他搜索是真的慢，不知道是我梯子原因，还是ai对于这种简单的操作有着复杂的思路问题，就是非常低效。这我也不说啥了，最后还给我漏掉好几个，其实我这个项目目录空文件是非常多的，实际写东西的也就一点点。后面，还是我手动一个一个把漏掉的手动补上了。待亏我写的东西不多，好维护。

# 小毛病
写着篇log时候，还发现了之前的放弃的smoking目录，一直存在里面，忘记删了。居然还不影响打包，还有就是改了图片储存目录后，ide不会自动识别本地的图片了，需要手动敲图片的名字。我想可能是vitepress框架底层写的有对于插入的路径有规定，自动会找public目录，再去找后面img目录，所以只需要写/img/xxx.png了，这里的第一个`/`就是`public`,但是ide是从本地电脑出发的，才不会关心你是什么框架，他只认识正常规范的目录路径啊，他的底层框架是操作系统！！！
